Mech Lords

Mechlords is a virtual reality multiplayer battle arena that pits two different player types against each other. The Overlord player faces off against 1-3 Mech Warriors in a battle to destroy each others base. The Mech Warriors have a first person view of the battle from their cockpits. The Overlord has a top down view of the arena, where he can deploy varying units, and give his units waypoints and targets.

Explore your cockpit in virtual reality with built in HUD and responsive avatar!

Input

A Leap Motion Controller tracked the position of the gun while a Wii Mote provided user input and rotational information

Enemy Behaviour

Enemies were driven by a simple finite state machine. Enemies move between searching and patrolling while the player's location is unknown. Once a player is seen an enemy will give chase. If the enemy loses sight of the player he will move to the players last known location and begin searching again. If an enemy catches a player it's game over, the player dies and restarts from the beginning of the level.

Unity Tutorials

Unity Tutorial Series: Modular Code and Custom Inspectors: Creating a Simple, Usable Tool.

I am creating a tutorial series designed to teach a design philosophy that has roots in aspect-oriented programming. The goal is to create simple, modular classes called Mixins. Mixins can work together to create a complex and customisable system. I teach this concept by using this philosophy to create a weapon and inventory system. The system we create uses Mixins to add new behaviours to the weapons. This allows the user to create many weapons with unique behaviours by combining these Mixins in different ways. This also allows the user to use these behaviours in other systems in their project, and add new behaviours to the existing system with ease. I then build a custom inspector for the class holding these behaviours, in order to display all behaviours and their parameters in a reorderable list.

Compete over falling food on spinning platforms to become the largest Hippo!

Hungry Hungry Tippos was the result of a week long game jam held at Sheridan College known as Sprint Week. Using the HappyFunTimes local multiplayer system we created a zany platformer where the heaviest hippo wins. Platforms spin easier beneath heavier hippos, so gaining weight can be powerful and dangerous! Being heavier gives you more control of the play area, but exposing you to the danger of being quickly tipped off the map. Also, smaller platforms will give way under too much weight.

I was the lead programmer on the team and was responsible for implementing all game mechanics and animations. We were provided a default controller from HappyFunTimes. This controller was Javascript based, allowing the user to visit a website on their mobile to join the game. I added to his controller, creating a hat selection menu. The selection menu was automatically populated by the number of hat sprites in the images directory. The chosen hats information was passed to Unity and placed on the respective character.
